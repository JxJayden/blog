起源

Go 语言起源 2007 年，并于 2009 年正式对外发布。它从 2009 年 9 月 21 日开始作为谷歌公司 20% 兼职项目，即相关员工利用 20% 的空余时间来参与 Go 语言的研发工作。该项目的三位领导者均是著名的 IT 工程师：Robert Griesemer，参与开发 Java HotSpot 虚拟机；Rob Pike，Go 语言项目总负责人，贝尔实验室 Unix 团队成员，参与的项目包括 Plan 9，Inferno 操作系统和 Limbo 编程语言；Ken Thompson，贝尔实验室 Unix 团队成员，C 语言、Unix 和 Plan 9 的创始人之一，与 Rob Pike 共同开发了 UTF-8 字符集规范。自 2008 年 1 月起，Ken Thompson 就开始研发一款以 C 语言为目标结果的编译器来拓展 Go 语言的设计思想。

代理与反射是什么？

通过调用 new Proxy() ，你可以创建一个代理用来替代另一个对象（被称为目标），这个代理对目标对象进行了虚拟，因此该代理与该目标对象表面上可以被当作同一个对象来对待。
代理允许你拦截在目标对象上的底层操作，而这原本是 JS 引擎的内部能力。拦截行为使用了一个能够响应特定操作的函数（被称为陷阱）。

色觉是有限的

人类的色彩感知有强弱

  1. 视觉是为了检查反差（边缘）优化的，而不是绝对亮度。 => 视觉系统对颜色和亮度的差别要敏感一些（对比鲜明的边缘）
  2. 辨别颜色的能力依赖于颜色是如何呈现的。=> 深浅度 & 色块的大小 & 分隔的距离 => 影响人类区分颜色的能力
  3. 有些人是色盲 => 使用的颜色能否被常见类型的色盲用户区分开
  4. 用户的屏幕和观看条件会影响对颜色的感知 => 彩色屏幕的差异 & 灰度显示器 & 显示器角度 & 环境光线

色觉的工作原理

视杆细胞（可以察觉光线强度，但感觉不到颜色） & 视椎细胞（可以察觉颜色） => 视杆细胞大多数时间出于过曝状态，视觉完全基于视椎细胞提供的信息。

三种视椎细胞敏感的光谱互相重叠

  1. 低频：对整个可见光频谱敏感，对于处于频谱中间的黄色和低频的红色最敏感
  2. 中频：对高频的蓝色和中频偏低的黄色有反应
  3. 高频：对可见光的高频部分（紫色 & 蓝色）最敏感，此类视椎细胞的整体敏感度较低，数量更少，所以人类对蓝色和紫色不如对其他颜色敏感。

如何看到各种颜色：“红-绿”减影信号通道，“黄-蓝”减影信号通道，“黑-白”亮度信号通道 => 颜色对抗通道

使用颜色的准则

  1. 用饱和度、亮度和色相来区分颜色 => 避免使用轻微的差别，确保色彩之间有较高的反差。
  2. 使用独特的颜色
  3. 避免使用色盲的人无法区分的颜色对
  4. 在颜色之外使用其他提示
  5. 将强烈的对抗色分开

糟糕的边界视力

中央凹的分辨率与视野边界的分辨率比较

边界是觉得信息在被传递到大脑之前是经过压缩的，而中央凹的视觉信息则没有。

眼睛以每秒三次的速度快速移动，选择性地将焦点投射到 周围的环境物体上，然后大脑基于我们所知和所期待的，填充视野的其他部分。

边界视觉的作用

  1. 提供低分辨率的线索
  2. 它可以很好的观察运动

让信息可见的常用方法

  1. 放在用户所看的位置
  2. 标记出错误
  3. 使用错误符号
  4. 保留红色以呈现错误
  5. 弹出式对话框中的信息 bad
  6. 使用声音 bad
  7. 闪烁或者短暂的晃动 bad

记忆力

传统观点 => 短期记忆和长期记忆分开
现代观点 => 短期和长期记忆用的是同一个记忆系统实现的。

长期记忆

记忆的形成是由参与某个神经活动模式的神经元长期甚至永久的变化组成。
